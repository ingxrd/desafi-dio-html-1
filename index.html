Mas antes, um pouco de história.

Compreender a história dos computadores permite observar o avanço e desenvolvimento da tecnologia, bem como a perceber que nós, seres humanos, sempre buscamos uma ajudinha das máquinas para resolver os nossos problemas. Por essa razão, antes de entrarmos na tela do VS Code e digitarmos nossas primeiras tags, te pergunto: qual a primeira imagem que lhe vem a cabeça quando dizemos computador? Uma tela preta, cheia de zeros e uns? Talvez o mouse, teclado, cpu, ou os sistemas operacionais como o Windows? Mas é fato que dificilmente relacionamos a ideia de computador com algo simples e distante da nossa época.

Entretanto, tudo tem um início. E no caso dos computadores, esse início remonta datas a.C!. O ábaco, por exemplo, ainda que manual e simples, foi criado com o objetivo de processar informações e realizar cálculos (isso não te lembra algo?). Isso ilustra muito bem que, para chegarmos no dia de hoje, onde linhas de código conseguem executar comandos, andamos um caminho gigantesco e por termos muita história nessa longa tragetória, faremos um recorte histórico.

O século XX foi conhecido pelos seus conflitos: duas guerras mundiais ocorreram em um curto períogo de tempo e os anos que se seguiram foram marcados pela Guerra Fria e contendas regionais. É nesse contexto histórico que a tecnologia encontrou terreno fértil para se desenvolver, devido a necessidade de inovações que impulsionassem os esforços de guerra e a competição entre as potências. Em 1946, a Alemanha desenvolveu o computador COLOSSUS para cálculos de guerra e decifração de mensagens alemãs codificadas. No mesmo ano, os Estados Unidos introduziram o ENIAC, projetado par cálculos balísticos e científicos.

É interessante pensar que esses computadores pesavam toneladas e hoje podemos carregá-los em nossas mãos. Com o passar dos anos, fomos introduzindo inovações, como o circuito integrado com transitores, capacitadores e resistores, que permitiu a redução de tamanho e peso dos computadores. Ao largo do tempo, os computadores deixaram de ser exclusividade dos militares e a introdução dos personal computers nos anos 70 (PCs) popularizou de vez o seu uso.

Ao passo em que os computadores se desenvolviam, a internet também era concebida e evoluía. Nos anos 1960, o projeto ARPANET, financiado pelo Departamento de Defesa dos Estados Unidos, foi o embrião da internet moderna. Seu objetivo inicial era criar uma rede de comunicação robusta e descentralizada, capaz de sobreviver a possíveis ataques durante a Guerra Fria. Com o tempo, essa rede se expandiu, conectando universidades e centros de pesquisa, e posteriormente se tornou a base da internet que conhecemos hoje.

Você deve ter percebido o quanto evoluimos desde o século passado e a velocidade com a qual concebemos novas ideias. Portanto, vamos deixar um pouco a história de lado e vamos focar em compreender uma parcela desse vasto mundo da computação.

O que são Clients e Servers?

Clients e Servers, ou no bom português, Clientes e Servidores são duas partes fundamentais do funcionamento da comunicação entre computadores. Vamos observar o esquema abaixo para nos ajudar a compreender a função de cada um.

Como discutimos anteriormente, a internet é uma rede de comunicação fundamental. É crucial compreender os participantes desse diálogo digital. Quem se comunica com quem? Os dispositivos que acessam conteúdo na internet são denominados Clientes. Em suma, são você, eu, qualquer pessoa que se conecta à internet usando seu dispositivo preferido, seja notebook, celular, tablet, entre outros. Utilizamos navegadores como Chrome, Internet Explorer, Firefox, entre outros, para acessar o conteúdo disponível na web. Quando digitamos algo e pressionamos 'enter', a internet interage com o Servidor, isto é, o servidor, que nos fornece as informações solicitadas, caso as tenha armazenado.

  • Servidores Web: Responsáveis por hospedar sites e páginas da web. O Apache e o Nginx são exemplos de servidores web populares.
  • Servidores de Banco de Dados: Responsáveis por armazenar e gerenciar dados. Exemplos incluem MySQL, PostgreSQL, MongoDB, entre outros.
  • Servidores de Arquivos: Utilizados para armazenamento e compartilhamento de arquivos em rede.
  • Servidores de Aplicativos: Usados para executar aplicativos e serviços. O servidor de aplicativos Java EE (Enterprise Edition) é um exemplo.
  • Servidores de E-mail: Gerenciam o envio, recebimento e armazenamento de e-mails. Exemplos incluem o Exchange Server da Microsoft e o Postfix.

Mão na massa: compreendendo a estrutura básica do HTML

Antes de iniciar a prática do HTML, será necessário um ambiente de desenvolvimento. Recomendamos o uso do VS Code, para baixar clique aqui.

Agora podemos falar sobre a estrutura básica do HTML, que é composta por três elementos:

  1. < html> : responsável por definir que nossa página é um HTML.
  2. < head> : que são as informações carregadas antes do usuário ver, como o título de uma página, localizado na aba.
  3. < body> : tudo que o usuário vê está entre as tags body.

Perceba que todas essas tags possuem fechamento, representado pelo /. Entretanto, algumas tags não precisam de fechamento, elas são vazias, como < / input type"text/">. Observe que ele fecha em si mesmo.

Além disso, também temos os atributos de tags, isto é, as propriedades associadas à elas.

Alguns exemplos de atributos são:
  • id:identificador para mudanças no Javascript.
  • style: Identificador para mudanças no CSS.
  • class: No CSS, podemos aplicar estilo a todos os elementos selecionados. Por exemplo: < class = "titulo-principal">. Dessa forma, todo títuli principal terá a mesma característica.

Agora que sabemos qual a estrutura do HTML, vamos aprender algumas tags úteis para fazer nosso primeiro site.

  • < h1> - < h6>: Define títulos ou cabeçalhos, onde < h1> é o mais importante e < h6> o menos importante.
  • < p>: Define parágrafos de texto.
  • < a>: Cria links para outras páginas ou recursos.
  • < img>: Insere imagens na página.
  • < ul>, < ol>, < li>: Criam listas não ordenadas (< ul>) e ordenadas (< ol>), com elementos de lista (< li>).

Colocando seu site para jogo: o que é hospedagem?

Agora que você sabe como usar tags em HTML, que tal publicar seu primeiro site? Para isso, precisamos hospedá-lo.

Hospedar um site significa, basicamente, alugar um servidor fornecido por uma empresa para armazenar os dados do nosso site. Isso permite que as pessoas acessem todo o conteúdo que criamos.
